<div id="installing-the-cli">
  ## Installation der CLI
</div>

<Info>
  **Voraussetzung**: Bitte installieren Sie [Node.js](https://nodejs.org/en), bevor Sie fortfahren.
</Info>

<Steps>
  <Step title="CLI installieren.">
    Führen Sie den folgenden Befehl aus, um die [CLI](https://www.npmjs.com/package/mint) zu installieren:

    <CodeGroup>
      ```bash npm
      npm i -g mint
      ```

      ```bash pnpm
      pnpm add -g mint
      ```
    </CodeGroup>
  </Step>

  <Step title="Lokal eine Vorschau starten.">
    Navigieren Sie zu Ihrem Docs-Verzeichnis (in dem sich Ihre `docs.json`-Datei befindet) und führen Sie den folgenden Befehl aus:

    ```bash
    mint dev
    ```

    Eine lokale Vorschau Ihrer Dokumentation ist unter `http://localhost:3000` verfügbar.
  </Step>
</Steps>

Alternativ können Sie, wenn Sie die CLI nicht global installieren möchten, ein einmaliges Skript ausführen:

```bash
npx mint dev
```

<div id="updates">
  ## Updates
</div>

Wenn Ihre lokale Vorschau nicht mit der Produktionsversion im Web übereinstimmt, aktualisieren Sie Ihre lokale CLI:

```bash
mint update
```

Wenn der Befehl `mint update` in Ihrer lokalen Version nicht verfügbar ist, installieren Sie die CLI in der neuesten Version neu:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash pnpm
  pnpm add -g mint@latest
  ```
</CodeGroup>

<div id="custom-ports">
  ## Benutzerdefinierte Ports
</div>

Standardmäßig verwendet die CLI Port 3000. Sie können den Port mit dem Flag `--port` anpassen. Um die CLI beispielsweise auf Port 3333 auszuführen, verwenden Sie diesen Befehl:

```bash
mint dev --port 3333
```

Wenn Sie versuchen, auf einem bereits belegten Port zu laufen, wird automatisch der nächste freie Port verwendet:

```mdx
Port 3000 ist bereits belegt. Verwende stattdessen 3001.
```

<div id="previewing-as-a-specific-group">
  ## Vorschau als bestimmte Gruppe
</div>

Wenn Sie partielle Authentifizierung verwenden, um den Zugriff auf Ihre Dokumentation einzuschränken, können Sie mit dem Flag `--group [groupname]` eine Vorschau als bestimmte Authentifizierungsgruppe anzeigen.

Wenn Sie beispielsweise eine Gruppe namens `admin` haben, können Sie als Mitglied dieser Gruppe mit folgendem Befehl eine Vorschau anzeigen:

```bash
mint dev --group admin
```

<div id="additional-commands">
  ## Zusätzliche Befehle
</div>

`mint dev` ist zwar der am häufigsten verwendete Befehl, aber es gibt weitere Befehle, mit denen Sie Ihre Dokumentation verwalten können.

<div id="finding-broken-links">
  ### Kaputte Links finden
</div>

Die CLI unterstützt Sie beim Validieren von Referenzlinks in Ihrer Dokumentation. Um kaputte Links zu erkennen, verwenden Sie den folgenden Befehl:

```bash
mint broken-links
```

<div id="checking-openapi-spec">
  ### OpenAPI-Spezifikation prüfen
</div>

Sie können die CLI verwenden, um Ihre OpenAPI-Datei mit dem folgenden Befehl auf Fehler zu prüfen:

```bash
mint openapi-check <openapiDateinameOderUrl>
```

Sie können einen Dateinamen (zum Beispiel `./openapi.yaml`) oder eine URL (zum Beispiel `https://petstore3.swagger.io/api/v3/openapi.json`) übergeben.

<div id="renaming-files">
  ### Dateien umbenennen
</div>

Sie können Dateien umbenennen und alle Verweise darauf mit dem folgenden Befehl aktualisieren:

```bash
mint rename <alterDateiname> <neuerDateiname>
```

<div id="migrating-mdx-endpoint-pages">
  ### Migrieren von MDX-Endpoint-Seiten
</div>

Sie können MDX-Endpoint-Seiten mit dem folgenden Befehl aus Ihrer OpenAPI-Spezifikation in automatisch generierte Seiten migrieren:

```bash
mint migrate-mdx
```

Dieser Befehl konvertiert einzelne MDX-Endpoint-Seiten in automatisch generierte Seiten, die in Ihrer `docs.json` definiert sind, verschiebt MDX-Inhalte in die `x-mint`-Erweiterung Ihrer OpenAPI-Spezifikation und aktualisiert Ihre Navigation. Weitere Details finden Sie unter [Migrating from MDX](/de/api-playground/migrating-from-mdx).

<div id="formatting">
  ## Formatierung
</div>

Für die lokale Entwicklung empfehlen wir, Erweiterungen in deiner IDE zu verwenden, um `MDX`-Dateien zu erkennen und zu formatieren.

Wenn du Cursor, Windsurf oder VS Code nutzt, empfehlen wir die [MDX VS Code Extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) für Syntaxhervorhebung und [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) für die Codeformatierung.

Wenn du JetBrains verwendest, empfehlen wir das [MDX IntelliJ IDEA Plugin](https://plugins.jetbrains.com/plugin/14944-mdx) für Syntaxhervorhebung sowie die Einrichtung von [Prettier](https://prettier.io/docs/webstorm) für die Codeformatierung.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="Fehler: Das Modul „sharp“ konnte mit der darwin-arm64-Runtime nicht geladen werden">
    Das kann an einer veralteten Node-Version liegen. Versuchen Sie Folgendes:

    1. Entfernen Sie die aktuell installierte Version der mint CLI: `npm uninstall -g mint`
    2. Aktualisieren Sie Node.js.
    3. Installieren Sie die mint CLI erneut: `npm install -g mint`
  </Accordion>

  <Accordion title="Problem: Unbekannter Fehler">
    **Lösung**: Wechseln Sie ins Root-Verzeichnis Ihres Geräts und löschen Sie den Ordner `~/.mintlify`. Führen Sie anschließend erneut `mint dev` aus.
  </Accordion>

  <Accordion title="Fehler: permission denied">
    Das liegt an fehlenden Berechtigungen zum globalen Installieren von Node-Paketen.

    **Lösung**: Führen Sie `sudo npm i -g mint` aus. Sie werden nach dem Passwort gefragt, das Sie zum Entsperren Ihres Computers verwenden.
  </Accordion>

  <Accordion title="Die lokale Vorschau sieht nicht so aus wie meine Docs im Web">
    Das liegt wahrscheinlich an einer veralteten CLI-Version.

    **Lösung:** Führen Sie `mint update` aus, um die neuesten Änderungen zu erhalten.
  </Accordion>

  <Accordion title="mintlify vs. mint package">
    Wenn Sie Probleme mit dem CLI-Paket haben, führen Sie zuerst `npm ls -g` aus. Dieser Befehl zeigt, welche Pakete global auf Ihrem Rechner installiert sind.

    Wenn Sie npm nicht verwenden oder es nicht in der -g-Liste sehen, versuchen Sie mit `which mint` die Installation zu finden.

    Wenn sowohl ein Paket namens `mint` als auch eines namens `mintlify` installiert ist, sollten Sie `mintlify` deinstallieren.

    1. Deinstallieren Sie das alte Paket:

    ```bash
      npm uninstall -g mintlify
    ```

    2. Leeren Sie den npm-Cache:

    ```bash
      npm cache clean --force
    ```

    3. Installieren Sie das neue Paket:

    ```bash
    npm i -g mint
    ```
  </Accordion>
</AccordionGroup>
