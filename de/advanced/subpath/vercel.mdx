<div id="verceljson-file">
  ## vercel.json-Datei
</div>

Die Datei `vercel.json` legt fest, wie Ihr Projekt gebaut und bereitgestellt wird. Sie liegt im Stammverzeichnis Ihres Projekts und steuert verschiedene Aspekte der Bereitstellung, einschließlich Routing, Redirects, Header und Build-Einstellungen.

Wir verwenden die `rewrites`-Konfiguration, um Anfragen von Ihrer Hauptdomain zu Ihrer Dokumentation zu proxien.

Rewrites leiten eingehende Anfragen intern an andere destinations weiter, ohne die URL im Browser zu ändern. Wenn jemand `yoursite.com/docs` aufruft, ruft Vercel intern Inhalte von `your-subdomain.mintlify.dev/docs` ab, im Browser bleibt jedoch `yoursite.com/docs` sichtbar. Das unterscheidet sich von Redirects, die Nutzer auf eine andere URL umleiten.

Sie können den Subpfad beliebig anpassen, z. B. auf `/docs`, `/help` oder `/guides`. Außerdem können Sie tief verschachtelte Subpfade wie `/product/docs` verwenden.

<div id="repository-structure">
  ## Repository-Struktur
</div>

Ihre Dokumentationsdateien müssen innerhalb Ihres Repository so organisiert sein, dass sie der von Ihnen gewählten Subpfadstruktur entsprechen. Wenn Sie Ihre Dokumentation beispielsweise unter `yoursite.com/docs` bereitstellen möchten, erstellen Sie ein Verzeichnis `docs/` mit all Ihren Dokumentationsdateien.

<div id="configuration">
  ## Konfiguration
</div>

Um Ihre Dokumentation unter einem benutzerdefinierten Unterpfad mit Vercel zu hosten, fügen Sie die folgende Konfiguration zu Ihrer `vercel.json` hinzu. Dieses Beispiel verwendet `/docs`, aber Sie können ihn durch jeden beliebigen Unterpfad ersetzen:

```json
{
  "rewrites": [
    {
      "source": "/docs",
      "destination": "https://[subdomain].mintlify.dev/docs"
    },
    {
      "source": "/docs/:match*",
      "destination": "https://[subdomain].mintlify.dev/docs/:match*"
    }
  ]
}
```

* **`source`**: Das Pfadmuster auf Ihrer Domain, das die Umschreibung (Rewrite) auslöst.
* **`destination`**: Das Ziel, an das die Anfrage per Proxy weitergeleitet werden soll.
* **`:match*`**: Ein Wildcard, das alle Pfadsegmente nach Ihrem Unterpfad erfasst.

Weitere Informationen finden Sie unter [Configuring projects with vercel.json: Rewrites](https://vercel.com/docs/projects/project-configuration#rewrites) in der Vercel-Dokumentation.

<div id="using-external-proxies-with-vercel">
  ## Verwendung externer Proxys mit Vercel
</div>

Wenn Sie einen externen Proxy (wie Cloudflare oder AWS CloudFront) vor Ihrer Vercel-Bereitstellung einsetzen, müssen Sie ihn korrekt konfigurieren, um Konflikte mit der Domain-Verifizierung von Vercel und der Ausstellung von SSL-Zertifikaten zu vermeiden.

Eine fehlerhafte Proxy-Konfiguration kann verhindern, dass Vercel SSL-Zertifikate von Let's Encrypt ausstellt, und zu Fehlern bei der Domain-Verifizierung führen.

Weitere Informationen finden Sie unter [unterstützte Anbieter](https://vercel.com/guides/how-to-setup-verified-proxy#supported-providers-verified-proxy-lite) in der Vercel-Dokumentation.

<div id="required-path-allowlist">
  ### Erforderliche Pfad-Allowlist
</div>

Ihr externer Proxy muss den Datenverkehr zu diesen konkreten Pfaden zulassen, ohne ihn zu blockieren, umzuleiten oder stark zu cachen:

- `/.well-known/acme-challenge/*` - erforderlich für die Zertifikatsprüfung durch Let's Encrypt
- `/.well-known/vercel/*` - erforderlich für die Domain-Verifizierung bei Vercel
- `/mintlify-assets/_next/static/*` - erforderlich für statische Assets

Diese Pfade sollten unverändert direkt an Ihre Vercel-Deployment-Instanz durchgereicht werden.

<div id="header-forwarding-requirements">
  ### Anforderungen an die Header-Weiterleitung
</div>

Stellen Sie sicher, dass Ihr Proxy den `HOST`-Header korrekt weiterleitet. Ohne korrekte Header-Weiterleitung schlagen Verifizierungsanfragen fehl.

<div id="testing-your-proxy-setup">
  ### Testen Ihrer Proxy-Konfiguration
</div>

So überprüfen Sie, ob Ihr Proxy korrekt konfiguriert ist:

1. Prüfen Sie, ob `https://[yourdomain].com/.well-known/vercel/` eine Antwort zurückgibt.
2. Stellen Sie sicher, dass SSL-Zertifikate in Ihrem Vercel Dashboard korrekt bereitgestellt werden.
3. Vergewissern Sie sich, dass die Domain-Verifizierung erfolgreich abgeschlossen wird.
