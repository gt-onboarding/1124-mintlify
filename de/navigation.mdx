Die Eigenschaft [Navigation](de/settings#param-navigation) in `docs.json` steuert die Struktur und Informationshierarchie Ihrer Dokumentation.

Mit einer passenden Navigation-Konfiguration können Sie Ihre Inhalte so organisieren, dass Nutzer genau das finden, was sie suchen.

<div id="pages">
  ## Seiten
</div>

Seiten sind die grundlegendste Navigationskomponente. Seiten entsprechen den MDX-Dateien, aus denen Ihre Dokumentation besteht.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/pages-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/pages-dark.png" alt="" />

Im Objekt `navigation` ist `pages` ein Array, in dem jeder Eintrag den Pfad zu einer [Seite](de/pages) angeben muss.

```json
{
  "navigation": {
    "pages": [
      "settings",
      "pages",
      "navigation",
      "themes",
      "custom-domain"
    ]
  }
}
```

<div id="groups">
  ## Gruppen
</div>

Verwenden Sie Gruppen, um die Navigation in Ihrer Seitenleiste in Abschnitte zu gliedern. Gruppen können ineinander verschachtelt, mit Tags versehen und mit Icons gestaltet werden.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/groups-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/groups-dark.png" alt="" />

Im Objekt `navigation` ist `groups` ein Array, in dem jeder Eintrag ein Objekt mit den erforderlichen Feldern `group` und `pages` ist. Die Felder `icon`, `tag` und `expanded` sind optional.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Erste Schritte",
        "icon": "play",
        "expanded": false,
        "pages": [
          "quickstart",
          {
            "group": "Bearbeitung",
            "icon": "pencil",
            "pages": [
              "installation",
              "editor"
            ]
          }
        ]
      },
      {
        "group": "Inhalte schreiben",
        "icon": "notebook-text",
        "tag": "NEW",
        "pages": [
          "writing-content/page",
          "writing-content/text"
        ]
      }
    ]
  }
}
```

<div id="default-expanded-state">
  ### Standardmäßig erweiterter Zustand
</div>

Setzen Sie `expanded: true` bei einer Gruppe, damit sie in der Navigation-Sidebar standardmäßig ausgeklappt ist. Das ist nützlich, um wichtige Abschnitte hervorzuheben oder die Auffindbarkeit von key Inhalten zu verbessern.

```json
{
  "group": "Erste Schritte",
  "expanded": true,
  "pages": ["quickstart", "installation"]
}
```

<div id="tabs">
  ## Tabs
</div>

Tabs erstellen separate Abschnitte Ihrer Dokumentation mit eigenen URL-Pfaden. Tabs erzeugen oben in Ihrer Dokumentation eine horizontale Navigation, über die Nutzer zwischen den Abschnitten wechseln können.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/tabs-light.png" alt="" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/tabs-dark.png" alt="" />

Im Objekt `navigation` ist `tabs` ein Array, in dem jeder Eintrag ein Objekt ist, das ein `tab`-Feld erfordert und weitere Navigationsfelder wie Gruppen, Seiten, icon oder Links zu externen Seiten enthalten kann.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "API-Referenz",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "tab": "SDKs",
        "icon": "code",
        "pages": [
          "sdk/fetch",
          "sdk/create",
          "sdk/delete"
        ]
      },
      {
        "tab": "Blog",
        "icon": "newspaper",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="menus">
  ### Menüs
</div>

Menüs fügen einer Registrieren Dropdown-Navigationselemente hinzu. Verwenden Sie Menüs, um Nutzern zu helfen, direkt zu bestimmten Seiten innerhalb einer Registrieren zu gelangen.

Im `navigation`-Objekt ist `menu` ein Array, in dem jeder Eintrag ein Objekt ist, das ein `item`-Feld erfordert und weitere Navigationsfelder wie Gruppen, Seiten, icon oder Links zu externen Seiten enthalten kann.

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Entwickler-Tools",
        "icon": "square-terminal",
        "menu": [
          {
            "item": "API-Referenz",
            "icon": "rocket",
            "groups": [
              {
                "group": "Kern-Endpoints",
                "icon": "square-terminal",
                "pages": [
                  "api-reference/get",
                  "api-reference/post",
                  "api-reference/delete"
                ]
              }
            ]
          },
          {
            "item": "SDKs",
            "icon": "code",
            "description": "SDKs werden zur Interaktion mit der API verwendet.",
            "pages": [
              "sdk/fetch",
              "sdk/create",
              "sdk/delete"
            ]
          }
        ]
      }
    ]
  }
}
```

<div id="anchors">
  ## Anker
</div>

Anker fügen oben in Ihrer Seitenleiste persistente Navigationselemente hinzu. Verwenden Sie Anker, um Ihre Inhalte zu gliedern, schnellen Zugriff auf externe Ressourcen bereitzustellen oder auffällige Handlungsaufforderungen zu platzieren.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/anchors-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/anchors-dark.png" />

Im `navigation`-Objekt ist `anchors` ein Array, in dem jeder Eintrag ein Objekt mit einem erforderlichen `anchor`-Feld ist und weitere Navigationsfelder enthalten kann, z. B. Gruppen, Seiten, icon oder Links zu externen Seiten.

```json
{
  "navigation": {
    "anchors": [
      {
        "anchor": "Dokumentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "anchor": "API-Referenz",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "anchor": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

Für Anker, die ausschließlich auf externe Links verweisen, verwenden Sie das Schlüsselwort `global`. Anker in einem `global`-Objekt müssen ein `href`-Feld enthalten und dürfen nicht auf einen relativen Pfad zeigen.

Globale Anker sind besonders nützlich, um auf Ressourcen zu verlinken, die nicht Teil Ihrer Dokumentation sind, aber für Ihre Nutzer leicht zugänglich sein sollten, etwa ein Blog oder ein Support-Portal.

```json
{
  "navigation": {
    "global":  {
      "anchors": [
        {
          "anchor": "Community",
          "icon": "house",
          "href": "https://slack.com"
        },
        {
          "anchor": "Blog",
          "icon": "pencil",
          "href": "https://mintlify.com/blog"
        }
      ]
    },
    "tabs": /*...*/
  }
}
```

<div id="dropdowns">
  ## Dropdowns
</div>

Dropdowns befinden sich in einem aufklappbaren Menü oben in der Sidebar-Navigation. Jeder Eintrag in einem Dropdown verweist auf einen Abschnitt Ihrer Dokumentation.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/dropdowns-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/dropdowns-dark.png" />

Im `navigation`-Objekt ist `dropdowns` ein Array, dessen Einträge Objekte sind, die ein `dropdown`-Feld benötigen und weitere Navigationsfelder wie Gruppen, Seiten, icon oder Links zu externen Seiten enthalten können.

```json
{
  "navigation": {
    "dropdowns": [
      {
        "dropdown": "Dokumentation",
        "icon": "book-open",
        "pages": [
          "quickstart",
          "development",
          "navigation"
        ]
      },
      {
        "dropdown": "API-Referenz",
        "icon": "square-terminal",
        "pages": [
          "api-reference/get",
          "api-reference/post",
          "api-reference/delete"
        ]
      },
      {
        "dropdown": "Blog",
        "href": "https://external-link.com/blog"
      }
    ]
  }
}
```

<div id="openapi">
  ## OpenAPI
</div>

Integrieren Sie OpenAPI-Spezifikationen direkt in Ihre Navigationsstruktur, um API-Dokumentation automatisch zu generieren. Erstellen Sie eigene API-Bereiche oder platzieren Sie Endpoint-Seiten innerhalb anderer Navigationselemente.

Legen Sie auf beliebiger Ebene Ihrer Navigationshierarchie eine Standard-OpenAPI-Spezifikation fest. Untergeordnete Elemente übernehmen diese Spezifikation, sofern sie nicht ihre eigene definieren.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "API-Referenz",
        "openapi": "/path/to/openapi-v1.json",
        "pages": [
          "overview",
          "authentication",
          "GET /users",
          "POST /users",
          {
            "group": "Products",
            "openapi": "/path/to/openapi-v2.json",
            "pages": [
              "GET /products",
              "POST /products"
            ]
          }
        ]
      }
    ]
  }
}
```

Weitere Info zum Verweisen auf OpenAPI-Endpoint in deiner Doku findest du unter [OpenAPI setup](/de/api-playground/openapi-setup).

<div id="versions">
  ## Versionen
</div>

Teilen Sie Ihre Navigation in verschiedene Versionen auf. Versionen sind über ein Dropdown-Menü auswählbar.

<img className="block dark:hidden pointer-events-none" src="/images/navigation/versions-light.png" />

<img className="hidden dark:block pointer-events-none" src="/images/navigation/versions-dark.png" />

Im `navigation`-Objekt ist `versions` ein Array, dessen Einträge jeweils Objekte sind, die ein `version`-Feld erfordern und beliebige weitere Navigationsfelder enthalten können.

```json
{
  "navigation": {
    "versions": [
      {
        "version": "1.0.0",
        "groups": [
          {
            "group": "Erste Schritte",
            "pages": ["v1/overview", "v1/quickstart", "v1/development"]
          }
        ]
      },
      {
        "version": "2.0.0",
        "groups": [
          {
            "group": "Erste Schritte",
            "pages": ["v2/overview", "v2/quickstart", "v2/development"]
          }
        ]
      }
    ]
  }
}
```

<div id="nesting">
  ## Verschachtelung
</div>

Sie können beliebige Kombinationen aus Ankern, Tabs und Dropdowns verwenden. Die Komponenten lassen sich flexibel ineinander verschachteln, um die gewünschte Navigationsstruktur zu erstellen.

<CodeGroup>
  ```json Anchors
  {
    "navigation": {
      "anchors": [
        {
          "anchor": "Anchor 1",
          "groups": [
            {
              "group": "Group 1",
              "pages": [
                "some-folder/file-1",
                "another-folder/file-2",
                "just-a-file"
              ]
            }
          ]
        },
        {
          "anchor": "Anchor 2",
          "groups": [
            {
              "group": "Group 2",
              "pages": [
                "some-other-folder/file-1",
                "various-different-folders/file-2",
                "another-file"
              ]
            }
          ]
        }
      ]
    }
  }
  ```

  ```json Tabs
  {
    "navigation": {
      "tabs": [
        {
          "tab": "Tab 1",
  ```

<div id="breadcrumbs">
  ## Breadcrumbs
</div>

Breadcrumbs zeigen den vollständigen Navigationspfad oben auf Seiten. Einige Themes haben Breadcrumbs standardmäßig aktiviert, andere nicht. Sie können steuern, ob Breadcrumbs für Ihre Website aktiviert sind, indem Sie die Eigenschaft `styling` in Ihrer `docs.json` konfigurieren.

<CodeGroup>
  ```json Display full breadcrumbs
  "styling": {
    "eyebrows": "breadcrumbs"
  }
  ```

  ```json Display parent section only
  "styling": {
    "eyebrows": "section"
  }
  ```
</CodeGroup>

<div id="interaction-configuration">
  ## Interaktionskonfiguration
</div>

Steuern Sie über die Eigenschaft `interaction` in Ihrer `docs.json`, wie Nutzer mit Navigationselementen interagieren.

<div id="enable-auto-navigation-for-groups">
  ### Automatische Navigation für Gruppen aktivieren
</div>

Wenn ein Nutzer eine Navigationsgruppe aufklappt, navigieren einige Themes automatisch zur ersten Seite dieser Gruppe. Sie können das Standardverhalten eines Themes mit der Option `drilldown` überschreiben.

* Auf `true` setzen, um die automatische Navigation zur ersten Seite zu erzwingen, wenn eine Navigationsgruppe ausgewählt wird.
* Auf `false` setzen, um die Navigation zu unterbinden und beim Auswählen nur die Gruppe ein- oder auszuklappen.
* Nicht setzen, um das Standardverhalten des Themes zu verwenden.

<CodeGroup>
  ```json Force navigation
  "interaction": {
    "drilldown": true  // Navigiere beim Aufklappen eines Dropdowns automatisch zur ersten Seite
  }
  ```

  ```json Prevent navigation
  "interaction": {
    "drilldown": false // Nie navigieren, nur die Gruppe ein- oder zuklappen
  }
  ```
</CodeGroup>
