<div id="about-mcp-servers">
  ## Über MCP-Server
</div>

Das Model Context Protocol (MCP) ist ein offenes Protokoll, das standardisierte Verbindungen zwischen KI-Anwendungen und externen Diensten wie Dokumentationen ermöglicht. Mintlify generiert aus Ihrer Dokumentation und Ihren OpenAPI-Spezifikationen einen MCP-Server und bereitet Ihre Inhalte für das breitere KI-Ökosystem vor, in dem sich jeder MCP-Client (wie Claude, Cursor, Goose und andere) mit Ihrer Dokumentation und Ihren APIs verbinden kann.

Ihr MCP-Server stellt Tools bereit, mit denen KI-Anwendungen Ihre Dokumentation durchsuchen und mit Ihren APIs interagieren können.

<div id="accessing-your-mcp-server">
  ## Zugriff auf Ihren MCP-Server
</div>

<Note>
  MCP-Server können nur für öffentliche Dokumentationen generiert werden. Dokumentationen hinter Endbenutzer-Authentifizierung können nicht zur Servergenerierung herangezogen werden.
</Note>

Mintlify generiert automatisch einen MCP-Server für Ihre Dokumentation und hostet ihn unter Ihrer Dokumentations-URL mit dem Pfad `/mcp`. Zum Beispiel ist der MCP-Server von Mintlify unter `https://mintlify.com/docs/mcp` verfügbar.

Sie können die URL Ihres MCP-Servers in Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) anzeigen und kopieren.

Der Pfad `/mcp` ist für gehostete MCP-Server reserviert und kann nicht für andere Navigationselemente verwendet werden.

<div id="configuring-your-mcp-server">
  ## Konfigurieren Ihres MCP-Servers
</div>

Alle MCP-Server enthalten standardmäßig das Tool `search`, mit dem Nutzer Informationen aus Ihrer Doku in anderen Tools abfragen können.

Wenn Sie einen [Pro- oder Enterprise-Plan](https://mintlify.com/pricing?ref=mcp) haben, können Sie Endpoints aus Ihrer OpenAPI-Spezifikation als MCP-Tools bereitstellen.

Um Endpoints als MCP-Tools bereitzustellen, verwenden Sie das Objekt `mcp` innerhalb der Erweiterung `x-mint` auf Datei- oder Endpoint-Ebene. Beispielsweise umfasst der Mintlify-MCP-Server Tools zum Erstellen von Assistant-Chats, zum Abrufen von Statusupdates und zum Auslösen von Updates.

MCP-Server verfolgen einen Security-first-Ansatz, bei dem API-Endpoints nicht standardmäßig exponiert werden. Sie müssen Endpoints explizit aktivieren, um sie als MCP-Tools verfügbar zu machen. Exponieren Sie nur Endpoints, die für den öffentlichen Zugriff über KI-Tools sicher sind.

<div id="file-level-configuration">
  ### Konfiguration auf Dateiebene
</div>

Aktivieren Sie MCP standardmäßig für alle Endpoints in einer OpenAPI-Spezifikationsdatei und schließen Sie bestimmte Endpoints gezielt aus:

```json
{
  "openapi": "3.1.0",
  "x-mint": {
    "mcp": {
      "enabled": true
    }
  },
  // ...
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": false // Deaktiviert MCP für diesen Endpoint
          }
        },
        // ...
      }
    }
  }
}
```

<div id="endpoint-level-configuration">
  ### Konfiguration auf Endpoint-Ebene
</div>

Aktivieren Sie MCP für bestimmte Endpoints:

```json
{
  "paths": {
    "/api/v1/users": {
      "get": {
        "x-mint": {
          "mcp": {
            "enabled": true,
            "name": "get-users",
            "description": "Liste der Benutzer abrufen"
          },
          // ...
        }
      }
    },
    "/api/v1/delete": {
      "delete": {
        // Kein `x-mint: mcp`, daher wird dieser Endpoint nicht als MCP-Tool bereitgestellt
        // ...
      }
    }
  }
}
```

<div id="using-your-mcp-server">
  ## Verwendung Ihres MCP-Servers
</div>

Ihre Nutzer müssen Ihren MCP-Server mit ihren bevorzugten KI-Tools verbinden.

1. Stellen Sie die URL Ihres MCP-Servers öffentlich bereit.
2. Nutzer kopieren die URL Ihres MCP-Servers und fügen sie in ihren Tools hinzu.
3. Nutzer greifen über ihre Tools auf Ihre Dokumentation und API-Endpoints zu.

So können Sie Ihren Nutzern helfen, eine Verbindung zu Ihrem MCP-Server herzustellen:

<Tabs>
  <Tab title="Kontextmenü">
    Fügen Sie Optionen im [Kontextmenü](/de/ai/contextual-menu) hinzu, damit Ihre Nutzer von jeder Seite Ihrer Dokumentation aus eine Verbindung zu Ihrem MCP-Server herstellen können.

    | Option | Bezeichner | Beschreibung |
    | :----- | :--------- | :---------- |
    | **MCP-Server-URL kopieren** | `mcp` | Kopiert die URL Ihres MCP-Servers in die Zwischenablage des Nutzers. |
    | **Mit Cursor verbinden** | `cursor` | Installiert Ihren MCP-Server in Cursor. |
    | **Mit VS Code verbinden** | `vscode` | Installiert Ihren MCP-Server in VS Code. |
  </Tab>

  <Tab title="Claude">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und finden Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und die Schritte enthält, um sie mit Claude zu verbinden.

        1. Navigieren Sie in den Claude-Einstellungen zur Seite [Connectors](https://claude.ai/settings/connectors).
        2. Wählen Sie **Add custom connector**.
        3. Fügen Sie den Namen Ihres MCP-Servers und die URL hinzu.
        4. Wählen Sie **Add**.
        5. Wählen Sie bei der Verwendung von Claude die Schaltfläche für Anhänge (das Plus-Icon).
        6. Wählen Sie Ihren MCP-Server aus.
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Model Context Protocol-Dokumentation](https://modelcontextprotocol.io/docs/tutorials/use-remote-mcp-server#connecting-to-a-remote-mcp-server).
  </Tab>

  <Tab title="Claude Code">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und finden Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und den Befehl enthält, um sie mit Claude Code zu verbinden.

        ```bash
        claude mcp add --transport http <name> <url>
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Claude Code-Dokumentation](https://docs.anthropic.com/en/docs/claude-code/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="Cursor">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und finden Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und die Schritte enthält, um sie mit Cursor zu verbinden.

        1. Verwenden Sie <kbd>Command</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> (<kbd>Strg</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd> unter Windows), um die Befehlspalette zu öffnen.
        2. Suchen Sie nach „Open MCP settings“.
        3. Wählen Sie **Add custom MCP**. Dadurch wird die Datei `mcp.json` geöffnet.
        4. Konfigurieren Sie in `mcp.json` Ihren Server:

        ```json
        {
          "mcpServers": {
            "<your-mcp-server-name>": {
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [Cursor-Dokumentation](https://docs.cursor.com/en/context/mcp#installing-mcp-servers).
  </Tab>

  <Tab title="VS Code">
    <Steps>
      <Step title="Rufen Sie Ihre MCP-Server-URL ab.">
        Navigieren Sie zu Ihrem [Dashboard](https://dashboard.mintlify.com/products/mcp) und finden Sie Ihre MCP-Server-URL.
      </Step>

      <Step title="Veröffentlichen Sie Ihre MCP-Server-URL für Ihre Nutzer.">
        Erstellen Sie eine Anleitung für Ihre Nutzer, die Ihre MCP-Server-URL und die Schritte enthält, um sie mit VS Code zu verbinden.

        1. Erstellen Sie die Datei `.vscode/mcp.json`.
        2. Konfigurieren Sie in `mcp.json` Ihren Server:

        ```json
        {
          "servers": {
            "<your-mcp-server-name>": {
              "type": "http",
              "url": "<your-mcp-server-url>"
            }
          }
        }
        ```
      </Step>
    </Steps>

    Weitere Details finden Sie in der [VS Code-Dokumentation](https://code.visualstudio.com/docs/copilot/chat/mcp-servers).
  </Tab>
</Tabs>

<div id="example-connecting-to-the-mintlify-mcp-server">
  ### Beispiel: Verbindung mit dem Mintlify MCP-Server
</div>

Stellen Sie eine Verbindung mit dem Mintlify MCP-Server her, um mit der Mintlify API zu interagieren und unsere Dokumentation zu durchsuchen. So erhalten Sie präzisere Antworten zur Nutzung von Mintlify in Ihrer lokalen Umgebung und zeigen gleichzeitig, wie Sie Ihren Nutzerinnen und Nutzern helfen können, sich mit Ihrem MCP-Server zu verbinden.

<Tabs>
  <Tab title="Kontextmenü">
    Wählen Sie oben auf dieser Seite das Kontextmenü und anschließend **Mit Cursor verbinden** oder **Mit VS Code verbinden**, um den Mintlify MCP-Server mit der IDE Ihrer Wahl zu verbinden.
  </Tab>

  <Tab title="Claude">
    So verwenden Sie den Mintlify MCP-Server mit Claude:

    <Steps>
      <Step title="Mintlify MCP-Server zu Claude hinzufügen">

        2. Wählen Sie **Add custom connector**.
        3. Fügen Sie den Mintlify MCP-Server hinzu:

        * Name: `Mintlify`
        * URL: `https://mintlify.com/docs/mcp`

        4. Wählen Sie **Add**.
      </Step>

      <Step title="Zugriff auf den MCP-Server im Chat">
        1. Wählen Sie in Claude die Schaltfläche für Anhänge (das Plus-Icon).
        2. Wählen Sie den Mintlify MCP-Server aus.
        3. Stellen Sie Claude eine Frage zu Mintlify.
      </Step>
    </Steps>

    Testen Sie die Verbindung, indem Sie Folgendes ausführen:

    ```bash
    claude mcp list
    ```

<div id="authentication">
  ## Authentifizierung
</div>

Wenn Sie einen API-Endpoint für MCP aktivieren, berücksichtigt der Server die in Ihren OpenAPI-`securitySchemes` und `securityRequirement` definierten Authentifizierungsanforderungen. Alle Keys werden direkt vom Tool verarbeitet und weder von Mintlify gespeichert noch weiterverarbeitet.

Wenn ein Nutzer sein KI-Tool anweist, einen geschützten Endpoint aufzurufen, fordert das Tool in diesem Moment die erforderlichen Anmeldedaten vom Nutzer an.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<AccordionGroup>
  <Accordion title="MCP-Server zeigt nur das Such-Tool">
    Wenn Ihr MCP-Server trotz vorhandener OpenAPI-Spezifikation nur das Such-Tool bereitstellt:

    1. Überprüfen Sie, ob Ihre OpenAPI-Spezifikation gültig und erreichbar ist.
    2. Stellen Sie sicher, dass Sie MCP für bestimmte Endpoints ausdrücklich mit `x-mint.mcp.enabled: true` aktiviert haben.
    3. Prüfen Sie Ihre Deployment-Logs auf Fehler bei der OpenAPI-Verarbeitung.

    Wenn die OpenAPI-Verarbeitung fehlschlägt, läuft der Server zur Sicherstellung der Funktionalität nur mit dem Such-Tool weiter.
  </Accordion>

  <Accordion title="Authentifizierungsprobleme">
    Wenn Nutzer Authentifizierungsprobleme melden:

    1. Prüfen Sie, ob Ihre OpenAPI-Spezifikation korrekte `securitySchemes`-Definitionen enthält.
    2. Vergewissern Sie sich, dass aktivierte Endpoints mit den angegebenen Authentifizierungsmethoden funktionieren.
  </Accordion>

  <Accordion title="Tool-Beschreibungen fehlen oder sind unklar">
    Wenn AI-Tools Ihre API-Endpoints nicht effektiv nutzen:

    1. Fügen Sie Ihren Endpoints aussagekräftige `summary`- und `description`-Felder hinzu.
    2. Stellen Sie sicher, dass Parameternamen und Beschreibungen selbsterklärend sind.
    3. Verwenden Sie das MCP-Dashboard, um zu prüfen, wie Ihre Endpoints als Tools erscheinen.
  </Accordion>
</AccordionGroup>
