Claude Code ist ein agentisches Befehlszeilentool, das Sie bei der Pflege Ihrer Dokumentation unterstützt. Es kann neue Inhalte verfassen, bestehende Seiten prüfen und die Dokumentation aktuell halten.

Sie können Claude Code darauf trainieren, Ihre Dokumentationsstandards und Workflows zu verstehen, indem Sie Ihrem Projekt eine `CLAUDE.md`-Datei hinzufügen und diese im Laufe der Zeit verfeinern.

<div id="getting-started">
  ## Erste Schritte
</div>

**Voraussetzungen:**

* Aktives Claude-Abonnement (Pro, Max oder API-Zugang)

**Einrichtung:**

1. Claude Code installieren:

```bash
  npm install -g @anthropic-ai/claude-code
```

2. Navigiere zu deinem docs-Verzeichnis.
3. (Optional) Füge die Datei `CLAUDE.md` unten zu deinem Projekt hinzu.
4. Führe `claude` aus, um zu starten.


<div id="claudemd-template">
  ## CLAUDE.md-Vorlage
</div>

Speichere eine `CLAUDE.md`-Datei im Stammverzeichnis deines Docs-Verzeichnisses, damit Claude Code dein Projekt besser versteht. Diese Datei trainiert Claude Code auf deine Dokumentationsstandards, Präferenzen und Workflows. Weitere Informationen findest du in den Anthropic-Dokumenten unter [Manage Claude&#39;s memory](https://docs.anthropic.com/en/docs/claude-code/memory).

Kopiere diese Beispielvorlage oder passe sie an die Spezifikationen deiner Docs an:

```mdx
# Mintlify-Dokumentation

## Arbeitsbeziehung
- Du kannst Ideen hinterfragen - das kann zu besserer Dokumentation führen. Führe Quellen an und erkläre deine Begründung, wenn du das tust
- Frage IMMER nach Klarstellung, anstatt Annahmen zu treffen
- Lüge, rate oder erfinde NIEMALS Informationen

## Projektkontext
- Format: MDX-Dateien mit YAML-Frontmatter
- Konfiguration: docs.json für Navigation, Theme, Einstellungen
- Komponenten: Mintlify-Komponenten

## Inhaltsstrategie
- Dokumentiere gerade genug für den Benutzererfolg - nicht zu viel, nicht zu wenig
- Priorisiere Genauigkeit und Benutzerfreundlichkeit der Informationen
- Mache Inhalte wenn möglich zeitlos
- Suche nach vorhandenen Informationen, bevor du neue Inhalte hinzufügst. Vermeide Duplikate, es sei denn, es geschieht aus strategischen Gründen
- Überprüfe vorhandene Muster auf Konsistenz
- Beginne mit den kleinsten sinnvollen Änderungen

## docs.json

- Beziehe dich auf das [docs.json-Schema](https://mintlify.com/docs.json) beim Erstellen der docs.json-Datei und der Website-Navigation

## Frontmatter-Anforderungen für Seiten
- Titel: Klarer, beschreibender Seitentitel
- Beschreibung: Prägnante Zusammenfassung für SEO/Navigation

## Schreibstandards
- Du-Form („du")
- Voraussetzungen am Anfang von prozeduralen Inhalten
- Teste alle Code-Beispiele vor der Veröffentlichung
- Passe Stil und Formatierung vorhandener Seiten an
- Schließe sowohl grundlegende als auch erweiterte Anwendungsfälle ein
- Sprach-Tags auf allen Code-Blocks
- Alt-Text auf allen Bildern
- Relative Pfade für interne Links

## Git-Workflow
- Verwende NIEMALS --no-verify beim Committen
- Frage, wie uncommittete Änderungen behandelt werden sollen, bevor du beginnst
- Erstelle einen neuen Branch, wenn kein klarer Branch für Änderungen existiert
- Committe häufig während der Entwicklung
- Überspringe oder deaktiviere NIEMALS Pre-Commit-Hooks

## Nicht tun
- Frontmatter bei MDX-Dateien überspringen
- Absolute URLs für interne Links verwenden
- Ungetestete Code-Beispiele einschließen
- Annahmen treffen - frage immer nach Klarstellung
```


<div id="sample-prompts">
  ## Beispiel‑Prompts
</div>

Sobald Sie Claude Code eingerichtet haben, probieren Sie diese Prompts aus, um zu sehen, wie es bei gängigen Dokumentationsaufgaben helfen kann. Sie können diese Beispiele direkt kopieren und einfügen oder sie an Ihre spezifischen Anforderungen anpassen.

<div id="convert-notes-to-polished-docs">
  ### Notizen in ausgereifte Dokumente verwandeln
</div>

Verwandeln Sie Rohentwürfe in saubere Markdown-Seiten mit Komponenten und Frontmatter.

**Beispielprompt:**

```text wrap
Konvertieren Sie diesen Text in eine korrekt formatierte MDX-Seite: [Text hier einfügen]
```


<div id="review-docs-for-consistency">
  ### Dokumentation auf Konsistenz prüfen
</div>

Erhalten Sie Vorschläge zur Verbesserung von Stil, Formatierung und Nutzung von Komponenten.

**Beispielprompt:**

```text wrap
Überprüfen Sie die Dateien in docs/ und schlagen Sie Verbesserungen zur Konsistenz und Klarheit vor
```


<div id="update-docs-when-features-change">
  ### Dokumentation bei Funktionsänderungen aktualisieren
</div>

Halten Sie die Dokumentation aktuell, wenn sich Ihr Produkt weiterentwickelt.

**Beispielprompt:**

```text wrap
Unsere API benötigt jetzt einen Versionsparameter. Aktualisieren Sie unsere Dokumentation, um version=2024-01 in allen Beispielen einzuschließen
```


<div id="generate-comprehensive-code-examples">
  ### Umfassende Codebeispiele erstellen
</div>

Erstellen Sie mehrsprachige Beispiele mit Fehlerbehandlung.

**Beispielprompt:**

```text wrap
Erstellen Sie Codebeispiele für [Ihren API-Endpoint] in JavaScript, Python und cURL mit Fehlerbehandlung
```


<div id="extending-claude-code">
  ## Claude Code erweitern
</div>

Statt Claude Code nur manuell zu prompten, können Sie es auch in Ihre bestehenden Workflows integrieren.

<div id="automation-with-github-actions">
  ### Automatisierung mit GitHub Actions
</div>

Führen Sie Claude Code automatisch aus, wenn sich der Code ändert, um die Dokumentation aktuell zu halten. Sie können Dokumentationsreviews bei Pull Requests auslösen oder Beispiele aktualisieren, wenn API-Änderungen erkannt werden.

<div id="multi-instance-workflows">
  ### Workflows mit mehreren Instanzen
</div>

Verwenden Sie separate Claude-Code-Sitzungen für unterschiedliche Aufgaben – eine zum Verfassen neuer Inhalte und eine weitere für Prüfung und Qualitätssicherung. So bleiben Konsistenz und Qualität gewahrt und Probleme, die in einer einzelnen Sitzung übersehen werden könnten, werden erkannt.

<div id="team-collaboration">
  ### Teamzusammenarbeit
</div>

Teilen Sie Ihre überarbeitete `CLAUDE.md`-Datei mit Ihrem Team, um einheitliche Dokumentationsstandards für alle Mitwirkenden sicherzustellen. Teams entwickeln häufig projektspezifische Prompts und Workflows, die fester Bestandteil ihres Dokumentationsprozesses werden.

<div id="custom-commands">
  ### Benutzerdefinierte Befehle
</div>

Erstellen Sie wiederverwendbare Slash-Befehle in `.claude/commands/` für häufige, projektspezifische oder teambezogene Dokumentationsaufgaben.
