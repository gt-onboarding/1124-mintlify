---
title: 'Response-Felder'
description: 'API-Antwortwerte anzeigen'
---

Die `<ResponseField>`-Komponente dient dazu, die Rückgabewerte einer API zu definieren. Viele Dokumentationen verwenden `<ResponseField>` auch auf Seiten, wenn Typen aufgelistet werden müssen.

<ResponseField name="response" type="string" required>
  Beispiel für ein Response-Feld
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Ein ResponseField-Beispiel
</ResponseField>
```


<div id="props">
  ## Props
</div>

<ResponseField name="name" type="string" required>
  Der Name des Response-Werts.
</ResponseField>

<ResponseField name="type" type="string" required>
  Erwarteter Typ des Response-Werts – dies kann eine beliebige Zeichenfolge sein.
</ResponseField>

<ResponseField name="default" type="string">
  Der Standardwert.
</ResponseField>

<ResponseField name="required" type="boolean">
  „Pflichtfeld“ neben dem Feldnamen anzeigen.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Ob ein Feld deprecated ist oder nicht.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Labels, die vor dem Feldnamen angezeigt werden
</ResponseField>

<ResponseField name="post" type="string[]">
  Labels, die nach dem Feldnamen angezeigt werden
</ResponseField>
