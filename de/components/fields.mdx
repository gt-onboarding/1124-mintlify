Verwenden Sie Felder, um API-Parameter und -Antworten zu dokumentieren. Es gibt zwei Feldtypen: Parameterfelder und Antwortfelder.

<div id="parameter-field">
  ## Parameterfeld
</div>

Die Komponente `<ParamField>` wird verwendet, um Parameter für Ihre APIs oder SDKs zu definieren. Das Hinzufügen eines `ParamField` fügt automatisch einen [API Playground](/de/api-playground/overview) hinzu.

<ParamField path="param" type="string" required>
  Ein Beispiel für ein Parameterfeld
</ParamField>

```mdx
<ParamField path="param" type="string" required>
  Ein Beispiel für ein Parameterfeld
</ParamField>
```

<div id="properties">
  ### Eigenschaften
</div>

<ParamField body="query, path, body, or header" type="string">
  Gibt an, ob es sich beim Parameter um einen Query-, Pfad-, Body- oder Header-Parameter handelt, gefolgt vom Parameternamen.
</ParamField>

<ParamField body="type" type="string">
  Erwarteter Typ des Parameterwerts.

  Unterstützt `number`, `string`, `boolean`, `object`.

  Arrays können mit dem Suffix `[]` definiert werden, z. B. `string[]`.
</ParamField>

<ParamField body="required" type="boolean">
  Gibt an, ob der Parameter ein Pflichtfeld ist.
</ParamField>

<ParamField body="deprecated" type="boolean">
  Gibt an, ob der Parameter veraltet ist.
</ParamField>

<ParamField body="default" type="any">
  Standardwert, der verwendet wird, wenn der Request-Wert leer ist.
</ParamField>

<ParamField body="placeholder" type="string">
  Platzhaltertext für die Eingabe im Playground.
</ParamField>

<ParamField body="children" type="string">
  Beschreibung des Parameters (Markdown-aktiviert).
</ParamField>

<div id="response-field">
  ## Response-Feld
</div>

Die `<ResponseField>`-Komponente definiert die Rückgabewerte einer API.

<ResponseField name="response" type="string" required>
  Ein Beispiel für ein Response-Feld
</ResponseField>

```mdx
<ResponseField name="response" type="string" required>
  Ein ResponseField-Beispiel
</ResponseField>
```

<div id="properties">
  ### Eigenschaften
</div>

<ResponseField name="name" type="string" required>
  Der Name des Response-Werts.
</ResponseField>

<ResponseField name="type" type="string" required>
  Erwarteter Typ des Response-Werts. Dies kann eine beliebige Zeichenkette sein.
</ResponseField>

<ResponseField name="default" type="string">
  Der Standardwert.
</ResponseField>

<ResponseField name="required" type="boolean">
  Gibt an, ob die Response ein Pflichtfeld ist.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Gibt an, ob ein Feld veraltet ist.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Bezeichnungen, die vor dem Namen des Feldes angezeigt werden.
</ResponseField>

<ResponseField name="post" type="string[]">
  Bezeichnungen, die nach dem Namen des Feldes angezeigt werden.
</ResponseField>
