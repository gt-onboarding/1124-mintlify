Verwenden Sie die `Update`-Komponente, um Changelog-Einträge, Versionsupdates und Release Notes einheitlich formatiert anzuzeigen.

<Update label="Beispiel" description="v0.1.1">
  ## Beispiel-Update

  Sie können hier alles hinzufügen, z. B. einen Screenshot, einen Codeausschnitt oder eine Liste von Änderungen.

  <Frame>
    <img className="block" src="https://mintlify-assets.b-cdn.net/hero-light.png" alt="" />
  </Frame>

  ### Features

  * Responsives Design
  * Anker für jedes Update
  * Generierter RSS-Feed-Eintrag für jedes Update
</Update>

<div id="how-to-use">
  ## Verwendungshinweise
</div>

```mdx Update example
<Update label="2024-10-11" description="v0.1.0" tags={["Mintlify"]}>
  Dies ist ein Update mit einer Beschriftung, Beschreibung und einem Tag.
</Update>
```

Verwenden Sie mehrere `Update`-Komponenten, um [Änderungsprotokolle](/de/guides/changelogs) zu erstellen.

<div id="props">
  ## Props
</div>

<ResponseField name="label" type="string" required>
  Beschriftung für das Update. Wird links vom Update angezeigt und erzeugt einen Ankerlink. Beschriftungen sollten eindeutig sein.
</ResponseField>

<ResponseField name="tags" type="string[]">
  Tags für das Update. Werden als Filter im rechten Seitenbereich angezeigt.
</ResponseField>

<ResponseField name="description" type="string">
  Beschreibung des Updates. Wird unter der Beschriftung und den Tags angezeigt.
</ResponseField>
