<div id="add-an-asyncapi-specification-file">
  ## Eine AsyncAPI-Spezifikationsdatei hinzufügen
</div>

Bevor Sie Seiten für Ihre WebSockets erstellen, stellen Sie sicher, dass Sie ein gültiges AsyncAPI-Schemadokument im JSON- oder YAML-Format haben, das der [AsyncAPI-Spezifikation](https://www.asyncapi.com/docs/reference/specification/v3.0.0) entspricht. Ihr Schema muss der AsyncAPI-Spezifikation 3.0+ entsprechen.

<Tip>
  Um zu prüfen, ob Ihr AsyncAPI-Schema gültig ist, können Sie es in
  den [AsyncAPI Studio](https://studio.asyncapi.com/) einfügen
</Tip>

<div id="auto-populate-websockets-pages">
  ## Websocket-Seiten automatisch erstellen
</div>

Sie können in der Navigation Ihrer `docs.json` jedem Reiter oder jeder Gruppe ein `asyncapi`-Feld hinzufügen. Dieses Feld kann entweder den Pfad zu einem AsyncAPI-Schemadokument in Ihrem Docs-Repository, die URL eines gehosteten AsyncAPI-Schemadokuments oder ein Array von Links zu AsyncAPI-Schemadokumenten enthalten. Mintlify generiert automatisch eine Seite für jeden AsyncAPI-WebSocket-Kanal.

**Beispiele mit Tabs:**

<CodeGroup>
  ```json Local File {5}
  "navigation": {
    "tabs": [
      {
          "tab": "API Reference",
          "asyncapi": "/path/to/asyncapi.json"
      }
    ]
  }

  ```

  ```json Remote URL {5}
  "navigation": {
    "tabs": [
      {
          "tab": "API Reference",
          "asyncapi": "https://github.com/asyncapi/spec/blob/master/examples/simple-asyncapi.yml"
      }
    ]
  }
  ```
</CodeGroup>

**Beispiele mit Gruppen:**

```json {8-11}
"navigation": {
  "tabs": [
    {
      "tab": "AsyncAPI",
      "groups": [
        {
          "group": "WebSockets",
          "asyncapi": {
            "source": "/path/to/asyncapi.json",
            "directory": "api-reference"
          }
        }
      ]
    }
  ]
}
```

<Note>
  Das Feld directory ist optional. Wenn es nicht angegeben ist, werden die Dateien im
  Ordner **api-reference** des Docs-Repos abgelegt.
</Note>


<div id="channel-page">
  ## Kanal-Seite
</div>

Wenn Sie die Reihenfolge Ihrer Kanäle gezielter steuern oder nur auf einen einzelnen Kanal verweisen möchten, können Sie eine MDX-Datei mit dem Feld `asyncapi` im Frontmatter erstellen.

```mdx
---
title: "WebSocket-Kanal"
asyncapi: "/path/to/asyncapi.json channelName"
---
```
