<div id="overview">
  ## Überblick
</div>

Der API Playground ist eine interaktive Umgebung, in der Nutzer Ihre API‑Endpoints testen und erkunden können. Entwickler können API‑Anfragen erstellen, senden und Antworten anzeigen, ohne Ihre Dokumentation zu verlassen.

<Frame>
  <img src="/images/playground/API-playground-light.png" alt="API Playground für den Endpoint zum Auslösen eines Updates." className="block dark:hidden" />

  <img src="/images/playground/API-playground-dark.png" alt="API Playground für den Endpoint zum Auslösen eines Updates." className="hidden dark:block" />
</Frame>

Der Playground wird automatisch aus Ihrer OpenAPI‑Spezifikation oder Ihrem AsyncAPI‑Schema generiert, sodass alle Updates an Ihrer API automatisch im Playground widergespiegelt werden. Sie können auch manuell API‑Referenzseiten erstellen, nachdem Sie eine Basis‑URL und eine Authentifizierungsmethode in Ihrer `docs.json` definiert haben.

Wir empfehlen, Ihren API Playground aus einer OpenAPI‑Spezifikation zu generieren. Weitere Informationen zur Erstellung Ihres OpenAPI‑Dokuments finden Sie unter [OpenAPI Setup](/de/api-playground/openapi-setup).

<div id="getting-started">
  ## Erste Schritte
</div>

<Steps>
  <Step title="Fügen Sie Ihre OpenAPI‑Spezifikationsdatei hinzu.">
    <Info>
      Stellen Sie sicher, dass Ihre OpenAPI‑Spezifikationsdatei gültig ist, z. B. mit dem [Swagger Editor](https://editor.swagger.io/) oder der [Mint CLI](https://www.npmjs.com/package/mint).
    </Info>

    ```bash {3}
    /your-project
      |- docs.json
      |- openapi.json
    ```
  </Step>

  <Step title="Konfigurieren Sie `docs.json`.">
    Aktualisieren Sie Ihre `docs.json`, um auf Ihre OpenAPI‑Spezifikation zu verweisen. Fügen Sie einem Navigationselement die Eigenschaft `openapi` hinzu, um Ihre Doku automatisch mit Seiten für jeden im OpenAPI‑Dokument definierten Endpoint zu füllen.

    Dieses Beispiel erzeugt eine Seite für jeden in `openapi.json` definierten Endpoint und ordnet sie in Ihrer Navigation unter der Gruppe „API‑Referenz“ ein.

    ```json
    "navigation": {
      "groups": [
        {
          "group": "API reference",
          "openapi": "openapi.json"
        }
      ]
    }
    ```

    Um Seiten nur für bestimmte Endpoints zu erzeugen, führen Sie die Endpoints in der Eigenschaft `pages` des Navigationselements auf.

    Dieses Beispiel erzeugt Seiten nur für die Endpoints `GET /users` und `POST /users`. Um weitere Endpoint‑Seiten zu erzeugen, fügen Sie zusätzliche Endpoints zum Array `pages` hinzu.

    ```json
    "navigation": {
      "groups": [
          {
            "group": "API reference",
            "openapi": "openapi.json",
            "pages": [
              "GET /users",
              "POST /users"
            ]
          }
      ]
    }
    ```
  </Step>
</Steps>

<div id="customizing-your-playground">
  ## Anpassen Ihres Playgrounds
</div>

Sie können Ihren API Playground anpassen, indem Sie die folgenden Eigenschaften in Ihrer `docs.json` definieren.

<ResponseField name="playground" type="object">
  Konfigurationen für den API Playground.

  <Expandable title="playground" defaultOpen="True">
    <ResponseField name="display" type="&#x22;interactive&#x22; | &#x22;simple&#x22; | &#x22;none&#x22;">
      Der Anzeigemodus des API Playground.

      * `"interactive"`: Zeigt den interaktiven Playground an.
      * `"simple"`: Zeigt einen kopierbaren Endpoint ohne Playground an.
      * `"none"`: Zeigt nichts an.

      Standardmäßig `interactive`.
    </ResponseField>

    <ResponseField name="proxy" type="boolean" defaultOpen="True">
      Ob API‑Anfragen über einen Proxy‑Server geleitet werden. Standardmäßig `true`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="examples" type="object">
  Konfigurationen für die automatisch generierten API‑Beispiele.

  <Expandable title="examples" defaultOpen="True">
    <ResponseField name="languages" type="array of string">
      Beispielsprachen für die automatisch generierten API‑Snippets.

      Sprachen werden in der angegebenen Reihenfolge angezeigt.
    </ResponseField>

    <ResponseField name="defaults" type="&#x22;required&#x22; | &#x22;all&#x22;">
      Ob optionale Parameter in API‑Beispielen angezeigt werden. Standardmäßig `all`.
    </ResponseField>
  </Expandable>
</ResponseField>

<div id="example-configuration">
  ### Beispielkonfiguration
</div>

```json
{
 "api": {
   "playground": {
     "display": "interactive"
   },
   "examples": {
     "languages": ["curl", "python", "javascript"],
     "defaults": "erforderlich"
   }
 }
}
```

Dieses Beispiel konfiguriert den API Playground so, dass er interaktiv ist, mit Beispiel-Code-Snippets für cURL, Python und JavaScript. In den Snippets werden nur Pflichtfelder angezeigt.

<div id="custom-endpoint-pages">
  ### Benutzerdefinierte Endpoint-Seiten
</div>

Wenn Sie mehr Kontrolle über Ihre API-Dokumentation benötigen, verwenden Sie die `x-mint`-Erweiterung in Ihrer OpenAPI-Spezifikation oder erstellen Sie einzelne `MDX`-Seiten für Ihre Endpoints.

Beide Optionen ermöglichen Ihnen:

* Seitenmetadaten anpassen
* Zusätzliche Inhalte wie Beispiele hinzufügen
* Playground-Verhalten pro Seite steuern

Die `x-mint`-Erweiterung wird empfohlen, damit Ihre gesamte API-Dokumentation automatisch aus Ihrer OpenAPI-Spezifikation generiert und in einer Datei verwaltet wird.

Einzelne `MDX`-Seiten werden für kleine APIs empfohlen oder wenn Sie Änderungen seitenweise experimentell ausprobieren möchten.

Weitere Informationen finden Sie unter [x-mint-Erweiterung](/de/api-playground/openapi-setup#x-mint-extension) und [MDX-Einrichtung](/de/api-playground/mdx/configuration).
