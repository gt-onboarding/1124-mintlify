---
title: "SDK-Beispiele hinzufügen"
description: "Sprachspezifische Codebeispiele neben Ihren API-Endpunkten anzeigen, um Entwicklern zu zeigen, wie sie Ihre SDKs verwenden"
---

Wenn Ihre Nutzer mit Ihrer API über ein SDK statt direkt über eine Netzwerkanfrage interagieren, können Sie die Erweiterung `x-codeSamples` verwenden, um Ihrem OpenAPI-Dokument Codebeispiele hinzuzufügen und sie auf Ihren OpenAPI-Seiten anzuzeigen.

Diese Eigenschaft kann zu jeder Anfragemethode hinzugefügt werden und hat das folgende Schema.

<ParamField body="lang" type="string" required>
  Die Sprache des Codebeispiels.
</ParamField>

<ParamField body="label" type="string">
  Die Beschriftung für das Beispiel. Dies ist hilfreich, wenn Sie mehrere Beispiele für einen einzelnen Endpunkt bereitstellen.
</ParamField>

<ParamField body="source" type="string" required>
  Der Quellcode des Beispiels.
</ParamField>

Hier ist ein Beispiel für Codebeispiele für eine App zur Pflanzenverfolgung, die sowohl ein Bash-CLI-Tool als auch ein JavaScript-SDK bietet.

```yaml
paths:
  /plants:
    get:
      # ...
      x-codeSamples:
        - lang: bash
          label: Alle ungegossenen Pflanzen auflisten
          source: |
            planter list -u
        - lang: javascript
          label: Alle ungegossenen Pflanzen auflisten
          source: |
            const planter = require('planter');
            planter.list({ unwatered: true });
        - lang: bash
          label: Alle eingetopften Pflanzen auflisten
          source: |
            planter list -p
        - lang: javascript
          label: Alle eingetopften Pflanzen auflisten
          source: |
            const planter = require('planter');
            planter.list({ potted: true });
```
