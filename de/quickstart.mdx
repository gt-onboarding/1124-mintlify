Dieser Schnellstartleitfaden zeigt Ihnen, wie Sie Ihre Dokumentationssite in wenigen Minuten einrichten und veröffentlichen.

Nach Abschluss dieses Leitfadens haben Sie eine live geschaltete Dokumentationssite, die Sie anpassen und erweitern können.

<Info>
  **Voraussetzungen**: Bevor Sie beginnen, [erstellen Sie ein Konto](https://mintlify.com/start) und schließen Sie das Onboarding ab.
</Info>

<div id="getting-started">
  ## Erste Schritte
</div>

Nachdem Sie den Onboarding-Prozess abgeschlossen haben, wird Ihre Dokumentations-Website automatisch unter einer eindeutigen URL mit folgendem Format bereitgestellt:

```
https://<ihr-projektname>.mintlify.app
```

Finden Sie Ihre URL auf der Übersichtsseite Ihres [Dashboards](https://dashboard.mintlify.com/).

<Frame>
  <img src="/images/quickstart/mintlify-domain-light.png" alt="Mintlify Domain" className="block dark:hidden" />

  <img src="/images/quickstart/mintlify-domain-dark.png" alt="Mintlify Domain" className="hidden dark:block" />
</Frame>

Die URL Ihrer Site ist sofort verfügbar. Verwenden Sie diese URL für Tests und zum Austausch mit Ihrem Team, während Sie Ihre Docs‑Site einrichten.

<div id="install-the-github-app">
  ### GitHub-App installieren
</div>

Mintlify stellt eine GitHub-App bereit, die die Bereitstellung automatisiert, wenn du Änderungen in dein Repository pushst.

Installiere die GitHub-App, indem du der Onboarding-Checkliste oder den Anweisungen in deinem Dashboard folgst.

1. Navigiere zu **Settings** in deinem Mintlify-Dashboard.
2. Wähle **GitHub App** in der Seitenleiste.
3. Wähle **Install GitHub App**. Dadurch wird ein neuer Tab mit der Installationsseite der GitHub-App geöffnet.
4. Wähle die Organisation oder das Benutzerkonto aus, in dem du die App installieren möchtest.
5. Wähle die Repositories aus, die du verbinden möchtest.

<Frame>
  <img src="/images/quickstart/github-app-installation-light.png" alt="GitHub App Installation" className="block dark:hidden" />

  <img src="/images/quickstart/github-app-installation-dark.png" alt="GitHub App Installation" className="hidden dark:block" />
</Frame>

<Info>
  Aktualisiere die Berechtigungen der GitHub-App, wenn du deine Dokumentation in ein anderes Repository verschiebst.
</Info>

<div id="authorize-your-github-account">
  ### Autorisieren Sie Ihr GitHub-Konto
</div>

1. Navigieren Sie zu **Settings** in Ihrem Mintlify-Dashboard.
2. Wählen Sie **My Profile** in der Seitenleiste.
3. Wählen Sie **Authorize GitHub account**. Dadurch wird ein neuer Registrieren zur GitHub-Autorisierungsseite geöffnet.

<Info>
  Je nach Einstellungen Ihrer Organisation muss möglicherweise ein Admin Ihrer GitHub-Organisation Ihr Konto autorisieren.
</Info>

<div id="code-based-workflow">
  ## Codebasierter Workflow
</div>

Der codebasierte Workflow integriert sich in Ihre bestehende Entwicklungsumgebung und Git-Repositories. Dieser Workflow eignet sich besonders für technische Teams, die ihre Dokumentation gemeinsam mit dem Code verwalten möchten.

<div id="install-the-cli">
  ### CLI installieren
</div>

Um lokal mit deiner Dokumentation zu arbeiten, installiere die Command Line Interface (CLI) namens [mint](https://www.npmjs.com/package/mint), indem du diesen Befehl in deinem Terminal ausführst:

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

<Info>
  Du benötigst Node.js auf deinem Rechner. Wenn bei der Installation Probleme auftreten, schau dir den Troubleshooting-Guide an.
</Info>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Nachdem Sie Ihre Umgebung eingerichtet haben, können Sie mit der Bearbeitung Ihrer Dokumentationsdateien beginnen. Aktualisieren Sie zum Beispiel den Titel der Einführungsseite:

1. Öffnen Sie das Repository, das Sie während des Onboardings erstellt haben.
2. Öffnen Sie `index.mdx` und gehen Sie zum Anfang der Datei:

```mdx index.mdx
---
title: "Einführung"
description: "Dies ist die Einführung zur Dokumentation"
---
```

3. Aktualisieren Sie das Feld `title` auf „Hello World“.

```mdx index.mdx {2}
---
title: "Hello World"
description: "Das ist die Einführung in die Dokumentation"
---
```

<div id="preview-the-changes">
  ### Änderungen in der Vorschau anzeigen
</div>

Um die Änderungen lokal in der Vorschau anzuzeigen, führen Sie den folgenden Befehl aus:

```bash
mint dev
```

Die Vorschau ist unter `localhost:3000` verfügbar.

<div id="push-the-changes">
  ### Änderungen pushen
</div>

Wenn Sie bereit sind, Ihre Änderungen zu veröffentlichen, pushen Sie sie in Ihr Repository.

Mintlify erkennt die Änderungen automatisch, baut Ihre Dokumentation und deployt die Updates auf Ihrer Site. Überwachen Sie den Deployment‑Status in der Commit‑Historie Ihres GitHub‑Repositorys oder im [Dashboard](https://dashboard.mintlify.com).

Nach Abschluss des Deployments ist Ihr neuestes Update unter `<your-project-name>.mintlify.app` verfügbar.

<div id="web-editor-workflow">
  ## Webeditor-Workflow
</div>

Der Webeditor-Workflow bietet eine What-you-see-is-what-you-get-(WYSIWYG)-Oberfläche zum Erstellen und Bearbeiten von Dokumentation. Dieser Workflow eignet sich besonders für Nutzer, die in ihrem Webbrowser ohne zusätzliche lokale Entwicklungstools arbeiten möchten.

<div id="access-the-web-editor">
  ### Auf den Webeditor zugreifen
</div>

1. Melden Sie sich bei Ihrem [Dashboard](https://dashboard.mintlify.com) an.
2. Wählen Sie in der linken Seitenleiste **Editor** aus.

<Info>
  Wenn Sie die GitHub-App nicht installiert haben, werden Sie beim Öffnen des Webeditors aufgefordert, die App zu installieren.
</Info>

  <img alt="Der Mintlify-Webeditor im visuellen Editor-Modus" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
</Frame>

<div id="edit-the-documentation">
  ### Dokumentation bearbeiten
</div>

Im Webeditor kannst du in der Seitenleiste durch deine Dokumentationsdateien navigieren. Aktualisieren wir die Einführungsseite:

Finde und wähle `index.mdx` im Dateiexplorer aus.

Aktualisiere dann im Editor das Titel-Feld auf „Hello World“.

<Frame>
  <img alt="Bearbeitung im Webeditor" src="/images/quickstart/web-editor-editing-light.png" className="block dark:hidden" />

  <img alt="Bearbeitung im Webeditor" src="/images/quickstart/web-editor-editing-dark.png" className="hidden dark:block" />

<Tip>
  Der Editor bietet eine umfangreiche Sammlung an Formatierungswerkzeugen und Komponenten. Tippe im Editor <kbd>/</kbd>, um das Befehlsmenü zu öffnen und auf diese Werkzeuge zuzugreifen.
</Tip>

<div id="publish-your-changes">
  ### Änderungen veröffentlichen
</div>

Wenn Sie mit Ihren Bearbeitungen zufrieden sind, klicken Sie oben rechts auf **Publish**. Ihre Änderungen werden sofort auf Ihrer Dokumentationsseite veröffentlicht.

<Tip>
  Verwenden Sie Branches, um Änderungen über Pull Requests vor der Veröffentlichung auf Ihrer Live‑Site zu prüfen und zu begutachten.
</Tip>

Weitere Details zur Verwendung des Webeditors, einschließlich der Nutzung von Branches und Pull Requests zur Zusammenarbeit und zum Vorschauen von Änderungen, finden Sie in unserer [Webeditor-Dokumentation](/de/editor/getting-started).

<div id="adding-a-custom-domain">
  ## Hinzufügen einer benutzerdefinierten domain
</div>

Während Ihre `<your-project-name>.mintlify.app` Subdomain für Tests und Entwicklung gut funktioniert, bevorzugen die meisten Teams für produktive Dokumentation eine benutzerdefinierte domain.

Um eine benutzerdefinierte domain hinzuzufügen, navigieren Sie in Ihrem Dashboard zur Seite [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain).

<Frame>
  <img src="/images/quickstart/custom-domain-light.png" alt="Custom Domain" className="block dark:hidden" />

  <img src="/images/quickstart/custom-domain-dark.png" alt="Custom Domain" className="hidden dark:block" />
</Frame>

Geben Sie Ihre domain ein (zum Beispiel `docs.yourcompany.com`) und folgen Sie den Anweisungen, um die DNS-Einstellungen bei Ihrem domain-Anbieter zu konfigurieren.

<Table>
  | Record Type | Name | Value | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs (oder Subdomain) | cname.mintlify.app | 3600 |
</Table>

<Info>
  DNS-Änderungen können bis zu 48 Stunden zur Propagierung benötigen, auch wenn sie häufig deutlich schneller abgeschlossen sind.
</Info>

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

Wenn während der Einrichtung Probleme auftreten, prüfen Sie diese häufigen Lösungen:

<AccordionGroup>
  <Accordion title="Lokale Vorschau funktioniert nicht">
    Stellen Sie sicher, dass Node.js v19+ installiert ist und dass Sie den Befehl `mint dev` aus dem Verzeichnis ausführen, das Ihre `docs.json`-Datei enthält.
  </Accordion>

  <Accordion title="Änderungen werden auf der Live‑Site nicht angezeigt">
    Die Bereitstellung kann einige Minuten dauern. Überprüfen Sie Ihre GitHub Actions (für Codebasierte Workflows) oder die Bereitstellungsprotokolle im Mintlify-Dashboard, um sicherzustellen, dass keine Build-Fehler vorliegen.
  </Accordion>

  <Accordion title="Benutzerdefinierte Domain wird nicht verbunden">
    Vergewissern Sie sich, dass Ihre DNS-Einträge korrekt eingerichtet sind, und lassen Sie ausreichend Zeit für die DNS-Propagation (bis zu 48 Stunden). Sie können Tools wie [DNSChecker](https://dnschecker.org) verwenden, um Ihren CNAME-Eintrag zu überprüfen.
  </Accordion>
</AccordionGroup>
