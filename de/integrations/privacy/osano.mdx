Fügen Sie Folgendes zu Ihrer Datei `docs.json` hinzu, um den Cookie-Einwilligungsmanager [Osano](https://www.osano.com/) zu integrieren.

<CodeGroup>
  ```json Integrationsoptionen in docs.json
  "integrations": {
      "osano": "SOURCE"
  }
  ```

  ```json Beispiel
  "integrations": {
      "osano": "https://cmp.osano.com/2sUB2dqwqdkks/8dqwd-dwd86£-4a9b/osano.js"
  }
  ```
</CodeGroup>

Der Wert für `SOURCE` befindet sich als `src`-Wert im von Osano generierten Codeausschnitt. Er beginnt immer mit `https://cmp.osano.com/`.

```html Code snippet from Osano
<script src="https://cmp.osano.com/placeholder/placeholder/osano.js"/>
```


<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<Accordion
  title="Seiten laden im Strict-Compliance-Modus nicht"
>
  Wenn Ihre Dokumentationsseiten beim Einsatz von Osanos **Strict**-Compliance-Modus nicht korrekt geladen werden, müssen Sie die Domain von Mintlify auf die Allowlist setzen, damit Bilder und andere Assets geladen werden können.

  <Steps>
    <Step title="Zu Managed Rules navigieren">
      Gehen Sie in Ihrem Osano-Dashboard zu **Scripts** → **Managed Rules**.
    </Step>

    <Step title="Mintlify-Domain hinzufügen">
      Fügen Sie `.mintlify.app/` als Managed Rule hinzu.

      <Frame>
        <img src="/images/integrations/osano-managed-rule.png" alt="Osano Managed Rule" />
      </Frame>

      <Info>
      So wird sichergestellt, dass alle von Mintlify ausgelieferten Assets (einschließlich Bildern, Stylesheets und anderer Dokumentationsressourcen) als essenziell behandelt werden und auch dann geladen werden, wenn Osano nicht kategorisierte Inhalte von Drittanbietern blockiert.
      </Info>
    </Step>
  </Steps>
</Accordion>
