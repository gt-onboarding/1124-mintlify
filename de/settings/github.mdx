Mintlify verwendet eine GitHub App, um Ihre Dokumentation automatisch mit Ihrem GitHub-Repository zu synchronisieren.

<div id="installing-the-github-app">
  ## Installation der GitHub App
</div>

Installieren Sie die Mintlify GitHub App im GitHub-Konto oder in der Organisation, die Ihr Docs-Repository besitzt. Sie müssen Organisationsinhaber sein oder über Administratorrechte im Repository verfügen, um die App zu installieren. Wenn Ihnen die erforderlichen Berechtigungen fehlen, muss der Repository-Inhaber die Installationsanfrage genehmigen.

Installieren Sie die Mintlify GitHub App über Ihr [Dashboard](https://dashboard.mintlify.com/settings/organization/github-app).

<Tip>
Wir empfehlen, den Zugriff nur auf das Repository zu beschränken, in dem Ihre Dokumentation gehostet wird.
</Tip>

<div id="permissions">
  ## Berechtigungen
</div>

Wenn Sie die GitHub App installieren, werden Sie aufgefordert, die folgenden Berechtigungen zu erteilen:

Leseberechtigungen:

* `metadata`: Grundlegende Repository-Informationen

Lese- und Schreibberechtigungen:

* `checks`: Statuschecks für Pull Requests erstellen
* `code`: Dateiänderungen lesen, wenn Sie Commits in Ihre Docs-Branch machen
* `deployments`: Vorschau-Deployments für Pull Requests erstellen
* `pull requests`: Branches und Pull Requests aus dem Webeditor erstellen
* `workflows`: GitHub Actions für Docs-Deployments verwalten

<Info>
Die App greift nur auf Repositories zu, denen Sie explizit Zugriff gewähren. Wenn Sie Branch-Schutzregeln aktiviert haben, kann die App nicht direkt auf geschützte Branches pushen.
</Info>

<div id="managing-repository-access">
  ## Verwalten des Repository-Zugriffs
</div>

Bei der Installation unserer GitHub App können Sie entweder allen Ihren Repositories oder nur ausgewählten Zugriff gewähren. Wir empfehlen, den Zugriff nur für die Repositories zu erteilen, in denen sich Ihre Dokumentation befindet. Sie können diese Auswahl jederzeit in Ihren [GitHub App-Einstellungen](https://github.com/apps/mintlify/installations/new) ändern.

<div id="configuring-docs-source">
  ## Docs-Quelle konfigurieren
</div>

Ändern Sie die Organisation, das Repository oder die Branch, aus der Ihre Dokumentation erstellt wird, im Bereich [Git-Einstellungen](https://dashboard.mintlify.com/settings/deployment/git-settings) Ihres Dashboards.

<div id="troubleshooting">
  ## Fehlerbehebung
</div>

<div id="github-app-connection-issues">
  ### Verbindungsprobleme mit der GitHub App
</div>

Wenn Sie Probleme mit der GitHub App haben, kann das Zurücksetzen der Verbindung die meisten Probleme lösen.

<Steps>
  <Step title="Deinstallieren Sie die Mintlify App über GitHub.">
    1. Gehen Sie in GitHub zu [Installations](https://github.com/settings/installations) und wählen Sie **Configure** neben der Mintlify App. Scrollen Sie nach unten und wählen Sie **Uninstall**.
    2. Gehen Sie zu [Authorized GitHub Apps](https://github.com/settings/apps/authorizations) und wählen Sie **Revoke** neben der Mintlify App.
  </Step>
  <Step title="Installieren Sie die Mintlify App erneut.">
    1. Gehen Sie in Ihrem Mintlify Dashboard zu [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) und installieren Sie die GitHub App.
    2. Autorisieren Sie Ihr Konto im Bereich [My Profile](https://dashboard.mintlify.com/settings/account) Ihres Dashboards.
  </Step>
</Steps>

<div id="feedback-add-ons-are-unavailable">
  ### Feedback-Add-ons sind nicht verfügbar
</div>

Wenn du die Optionen für Bearbeitungsvorschläge oder Issue-Meldungen in deinem Dashboard nicht aktivieren kannst und dein Repository öffentlich ist, überprüfe deine Git-Einstellungen erneut.

<Steps>
  <Step title="Zu den Git-Einstellungen navigieren">
    Gehe in deinem Dashboard zu den [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings).
  </Step>
  <Step title="Einstellungen erneut überprüfen">
    Klicke auf das grüne Häkchen in der Ecke des Git-Settings-Felds, um die Repository-Einstellungen neu zu überprüfen. Dadurch werden deine Repository-Einstellungen aktualisiert, sodass korrekt erkannt wird, ob dein Repository öffentlich oder privat ist.
