<Info>
  [Pro- und Enterprise-Pläne](https://mintlify.com/pricing?ref=docs-ci) enthalten CI-Prüfungen für GitHub-Repositories.
</Info>

Verwenden Sie CI-Prüfungen, um Ihre Dokumentation zu linten, Fehler zu erkennen und vor der Bereitstellung Warnungen auszugeben. CI-Prüfungen sind so konfiguriert, dass sie bei Commits in Ihre konfigurierte Bereitstellungs-Branch oder bei Pull Requests gegen diese Branch ausgeführt werden.

<div id="installation">
  ## Installation
</div>

Beginnen Sie, indem Sie die Schritte auf der Seite [GitHub](/de/settings/github) ausführen.

Bei GitHub Apps können Sie Berechtigungen ausschließlich für ein einzelnes Repository vergeben. Wir empfehlen dies ausdrücklich, da wir nur Zugriff auf das Repository benötigen, in dem Ihre Dokumentation gehostet wird.

<div id="configuration">
  ## Konfiguration
</div>

Konfigurieren Sie die für eine Bereitstellung aktivierten CI-Prüfungen, indem Sie in Ihrem Dashboard zur Seite [Add-ons](https://dashboard.mintlify.com/products/addons) navigieren. Aktivieren Sie die Prüfungen, die Sie ausführen möchten.

Beim Aktivieren von Prüfungen können Sie festlegen, ob sie auf `Warning`- oder `Blocking`-Ebene ausgeführt werden.

* Eine Prüfung auf `Warning`-Ebene führt nie zu einem Fehlerstatus, selbst wenn Fehler oder Vorschläge vorliegen.
* Eine Prüfung auf `Blocking`-Ebene führt zu einem Fehlerstatus, wenn sie nicht bestanden wird oder Änderungen vorgeschlagen werden.

<div id="available-ci-checks">
  ## Verfügbare CI-Prüfungen
</div>

<div id="broken-links">
  ### Defekte Links
</div>

Ähnlich wie der [CLI-Link-Checker](/de/settings/broken-links#broken-links) auf deinem lokalen Rechner prüft, überprüfen wir deine Dokumentation automatisch auf defekte Links.

Um die Ergebnisse dieser Prüfung einzusehen, rufe auf GitHub die Ergebnisseite für den jeweiligen Commit auf.

<div id="vale">
  ### Vale
</div>

[Vale](https://vale.sh/) ist ein Open-Source‑, regelbasierter Prosa‑Linter, der verschiedene Dokumenttypen unterstützt, darunter Markdown und MDX.

Mintlify kann Vale automatisch in einem CI-Check ausführen und die Ergebnisse als Prüfstatus anzeigen.

<div id="configuration">
  #### Konfiguration
</div>

Wenn sich eine `.vale.ini`-Datei im Stammverzeichnis der Inhalte Ihrer Bereitstellung befindet, verwenden wir diese Konfigurationsdatei automatisch. Wir verwenden außerdem automatisch alle Konfigurationsdateien im angegebenen `stylesPath`.

Wenn keine Vale-Konfigurationsdatei vorhanden ist, wird die Standardkonfiguration automatisch geladen.

```mdx Default vale.ini configuration expandable
# Styles der obersten Ebene
StylesPath = /app/styles
MinAlertLevel = suggestion
IgnoredScopes = code, tt, img, url, a
SkippedScopes = script, style, pre, figure, code

# Vokabulare
Vocab = Mintlify

# Dies ist erforderlich, da Vale MDX nicht offiziell unterstützt
[formats]
mdx = md

# MDX-Unterstützung
[*.mdx]
BasedOnStyles = Vale
Vale.Terms = NO # Erzwingt sehr strenge Großschreibungsregeln, deaktiviert lassen

# `import ...`, `export ...`
# `<Component ... />`
# `<Component>...</Component>`
# `{ ... }`
TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

# Ausschließen:
# `<Component \n ... />`
BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})

CommentDelimiters = {/*, */}
```

<Warning>
  Bitte beachten Sie, dass wir aus Sicherheitsgründen keine absoluten `stylesPath`-Angaben oder `stylesPath`-Werte unterstützen, die `..` enthalten. Verwenden Sie relative Pfade und legen Sie den `stylesPath` in Ihrem Repository ab.
</Warning>

<div id="packages">
  #### Pakete
</div>

Vale unterstützt eine Reihe von [Paketen](https://vale.sh/docs/keys/packages), mit denen Sie Rechtschreib- und Stilfehler prüfen können.
Alle Pakete, die Sie in Ihrem Repository unter dem korrekten `stylesPath` hinzufügen, werden automatisch installiert und in Ihrer Vale-Konfiguration verwendet.

Für Pakete, die nicht in Ihrem Repository enthalten sind, können Sie beliebige Pakete aus dem [Vale-Paketregister](https://vale.sh/explorer) angeben; sie werden automatisch heruntergeladen und in Ihrer Vale-Konfiguration verwendet.

<Warning>
  Bitte beachten Sie, dass wir aus Sicherheitsgründen das automatische Herunterladen von Paketen, die nicht aus dem [Vale-Paketregister](https://vale.sh/explorer) stammen, nicht unterstützen können.
</Warning>

<div id="vale-with-mdx">
  #### Vale mit `MDX`
</div>

Vale unterstützt `MDX` nicht nativ, aber der Autor von Vale hat eine [benutzerdefinierte Erweiterung](https://github.com/errata-ai/MDX) dafür bereitgestellt.

Wenn Sie diese Erweiterung nicht verwenden möchten, empfehlen wir, die folgenden Zeilen in Ihrer Datei `.vale.ini` hinzuzufügen:

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Um die Inline-Kommentare von Vale zu verwenden, nutzen Sie MDX-Kommentare `{/* ... */}`. Wenn Sie in Ihrer Konfiguration die [Einstellung](https://vale.sh/docs/keys/commentdelimiters) `CommentDelimiters = {/*, */}` verwenden, interpretiert Vale diese Kommentare beim Linting automatisch. Das bedeutet, dass Sie die integrierten Funktionen von Vale – wie das Überspringen von Zeilen oder Abschnitten – ganz einfach nutzen können.

```mdx
{/* vale off */}

Dieser Text wird von Vale ignoriert

{/* vale on */}
```

Wenn Sie sich entscheiden, `CommentDelimiters` nicht zu verwenden, aber dennoch die Kommentare von Vale zu nutzen, müssen Sie alle Vale-Kommentare in MDX-Kommentare `{/* ... */}` einbetten. Zum Beispiel:

```mdx
{/* <!-- vale off --> */}

Dieser Text wird von Vale ignoriert

{/* <!-- vale on --> */}
```

Diese Kommentar-Tags werden innerhalb von Mintlify-Komponenten nicht unterstützt, können jedoch überall auf der Basisebene eines Dokuments verwendet werden.
