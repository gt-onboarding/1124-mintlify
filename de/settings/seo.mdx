Mintlify setzt viele SEO-Best Practices automatisch um, darunter:

* Generierung von Meta-Tags
* Erzeugung einer Sitemap und der Datei `robots.txt`
* Semantische HTML-Struktur
* Mobile Optimierung

Sie können die Meta-Tags Ihrer Website vollständig anpassen, indem Sie das Feld `metatags` zu Ihrer `docs.json` oder zum Frontmatter einer Seite hinzufügen.

<div id="global-meta-tags">
  ## Globale Meta-Tags
</div>

Sie können Standard-Meta-Tags festlegen, indem Sie das Feld `metadata` zu Ihrer `docs.json` hinzufügen.

```json
"seo": {
    "metatags": {
        "og:image": "Link zu Ihrem Standard-Meta-Tag-Bild"
    }
}
```

<div id="page-specific-meta-tags">
  ## Seitenspezifische Meta-Tags
</div>

Fügen Sie die Meta-Tags wie folgt in die Metadaten der Seite ein:

```mdx
---
title: 'Ihr Beispiel-Seitentitel'
'og:image': 'Link zu Ihrem Meta-Tag-Bild'
---
```

<Warning>
  Meta-Tags mit Doppelpunkten müssen in Anführungszeichen stehen, wenn du sie in die Seitenmetadaten einfügst.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Sitemaps und robots.txt-Dateien
</div>

Mintlify erstellt automatisch eine `sitemap.xml`-Datei sowie eine `robots.txt`-Datei. Sie können Ihre Sitemap anzeigen, indem Sie `/sitemap.xml` an die URL Ihrer Dokumentationsseite anhängen.

Standardmäßig werden nur Seiten berücksichtigt, die in Ihrer `docs.json` enthalten sind. Um versteckte Links einzubeziehen, fügen Sie `seo.indexing` zu Ihrer `docs.json` hinzu:

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Eigene Sitemaps und robots.txt-Dateien
</div>

Um eine eigene `sitemap.xml`- oder `robots.txt`-Datei hinzuzufügen, erstellen Sie eine `sitemap.xml`- oder `robots.txt`-Datei im Stammverzeichnis Ihres Projekts. Eine eigene Datei ersetzt die automatisch generierte Datei gleichen Namens. Wenn Sie die eigene Datei löschen, wird wieder die Standarddatei verwendet.

<div id="disabling-indexing">
  ## Deaktivieren der Indexierung
</div>

Wenn Sie verhindern möchten, dass eine Seite von Suchmaschinen indexiert wird, können Sie Folgendes im [Frontmatter](de/pages) Ihrer Seite einfügen:

```
---
noindex: true
---
```

Sie können auch `noindex` für alle Seiten in Ihrer Dokumentation festlegen, indem Sie das Feld `metadata.robots` in Ihrer `docs.json` auf `"noindex"` setzen:

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO-Best Practices
</div>

<AccordionGroup>
  <Accordion title="Aussagekräftige Titel und Beschreibungen verfassen">
    * Verwenden Sie klare, prägnante Seitentitel (50–60 Zeichen)
    * Schreiben Sie ansprechende Beschreibungen (150–160 Zeichen)
    * Integrieren Sie relevante Keywords
    * Machen Sie jeden Seitentitel und jede Beschreibung einzigartig
  </Accordion>

  <Accordion title="Inhaltsstruktur optimieren">
    * Verwenden Sie eine korrekte Überschriftenhierarchie (H1 → H2 → H3)
    * Schreiben Sie zuerst für Menschen, danach für Suchmaschinen
    * Platzieren Sie relevante Keywords in Überschriften und Inhalt
    * Halten Sie URLs kurz, aussagekräftig und hierarchisch organisiert
    * Strukturieren Sie längere Inhalte mit Zwischenüberschriften und Listen
  </Accordion>

  <Accordion title="Interne Verlinkungsstrategie">
    * Verlinken Sie auf themenverwandte Seiten innerhalb Ihrer Dokumentation
    * Verwenden Sie beschreibenden Linktext statt „Hier klicken“
    * Bilden Sie Themen-Cluster, indem Sie verwandte Inhalte verlinken
    * Nutzen Sie die automatischen Funktionen für Querverweise
  </Accordion>

  <Accordion title="Bild-SEO">
    * Verwenden Sie aussagekräftige Dateinamen für Bilder
    * Fügen Sie stets alt-Text für Barrierefreiheit und SEO hinzu
    * Optimieren Sie die Dateigröße von Bildern für schnellere Ladezeiten
    * Verwenden Sie relevante Bilder, die Ihren Inhalt unterstützen
  </Accordion>
</AccordionGroup>
